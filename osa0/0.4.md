sequenceDiagram
    participant browser
    participant server
    
    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note
    activate server
    server-->>browser: HTML document
    deactivate server
    
    Kun lomakkeen nappia painetaan, lähettää selain lomakkeelle syötetyn datan palvelimelle. Kyseessä on HTTP POST ‑pyyntö ja palvelin vastaa pyyntöön HTTP-statuskoodilla 302. Kyseessä on ns. uudelleenohjauspyyntö eli redirectaus, jolloin selain tekee automaattisesti uuden HTTP GET ‑pyynnön headerin Location viittaamaan paikkaan eli osoitteeseen notes.

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/notes
    activate server
    server-->>browser: HTML document
    deactivate server

    Selain hakee palvelimelta sivun sisällön HTTP GET-pyynnöllä
    
    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.css
    activate server
    server-->>browser: the css file
    deactivate server

    Selain hakee palvelimelta tyylitiedoston HTTP GET-pyynnöllä
    
    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.js
    activate server
    server-->>browser: the JavaScript file
    deactivate server

    Selain alkaa suorittamaan noudettua JavaScript-koodia, joka tekee http GET-pyynnön osoitteeseen https://studies.cs.helsinki.fi/exampleapp/data.json, josta muistiinpanot palautetaan JSON-muotoisena raakadatana
    
    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/data.json
    activate server
    server-->>browser: [{ "content": "HTML is easy", "date": "2023-1-1" }, ... ]
    deactivate server    

    Datan saapuessa selain suorittaa tapahtumankäsittelijän, joka renderöi muistiinpanot ruudulle käyttäen DOM-apia
